# 当前阶段与后续方向

## 当前状态

项目已从“基础模板可运行”推进到“主链路可用、后台标准页成型、开发工具可进入日常使用”的状态。

当前已经完成的主线包括：

- 参数设置中心收口为唯一系统设置入口
- 多角色权限模型落地
- 用户状态协议统一
- 后台列表页 / 树形页 / 日志页母版收口
- 操作日志与审计链恢复并增强
- 系统接口、表单构建、代码生成不再是空壳
- 本地质量检查与 Playwright 冒烟建立

## 当前最值得继续的方向

### 1. 先补平台能力层，再接首个真实业务模块

当前已完成平台底座层主线，但能力层仍偏弱。

根据 [platform-layer-audit.md](/Users/linshen/Desktop/Full-stack-TestV1/.ai-memory/platform-layer-audit.md) 和 [platform-capability-phase1.md](/Users/linshen/Desktop/Full-stack-TestV1/.ai-memory/platform-capability-phase1.md) 的结论，建议顺序调整为：

1. 审批流 MVP + 模块注册 / 模块入口配置最小版
2. 我的待办 / 我发起的 + 统一业务操作日志最小规范
3. 附件中心最小版
4. 再接入“财务预算管控模块”

当前 Step 1 已落地：

- `app/platform` 后端目录已建立
- 审批流核心表与模块注册表已新增
- 流程定义 / 节点定义 / 流程实例 / 审批任务 / 动作日志 / 业务绑定的核心后端接口已落地
- `finance-budget` 作为首个业务模块样板的注册元数据已入迁移与初始化 SQL

当前未完成：

- 统一业务操作日志最小规范
- 财务预算模块样板接入

当前 Step 2 前端最小验收层已落地：

- `我的待办` 页面
- `我发起的` 页面
- 流程实例详情抽屉
- 页面内最小审批动作：通过 / 驳回 / 撤回
- 为支撑前端验收，后端补了最小列表接口：
  - `GET /api/v1/platform/workflow/tasks/todo`
  - `GET /api/v1/platform/workflow/instances/started`

当前 Step 3 已落地并完成 live 联调：

- 后端附件中心最小版已新增：
  - `att_file`
  - 上传 / 列表 / 受控下载 / 删除接口
- 流程实例详情抽屉已接入最小附件区块
- Playwright 附件 smoke 已通过：
  - `e2e/workflow-attachment-smoke.spec.ts`

当前关于“待办 / 通知”的口径已定：

- `我的待办` 作为审批任务真相源
- 右上角提示图标保留为未来通知中心入口占位
- 在通知中心未建设前，不接真实待办数据，不使用待办数量作为提示数
- 当前前端表现已按该规则调整：
  - 保留右上角图标入口
  - 移除旧演示通知数据
  - 弹层仅展示“通知中心建设中”的占位说明

当前关于“头像下拉 / 个人中心”的口径已定：

- 右上角头像下拉只保留真实用户入口，不再展示框架演示菜单
- `个人中心` 必须稳定打开 `/profile`
- `问题&帮助` 当前只保留占位入口
- 默认头像使用“用户名首字母大写 + 账号级稳定纯色背景”
- 后续个人中心承接“自定义头像 / 默认背景色”设置能力
- 个人中心最小真实能力也已补齐：
  - 修改密码走真实接口
  - 个人简介可持久化更新
  - 角色、部门、岗位、权限继续只读展示

### 2. 审批流平台模块

建议做成：

- 流程定义
- 节点定义
- 流程实例
- 审批任务
- 动作日志
- 业务挂接

不建议第一版直接做重型 BPMN 设计器。

### 3. 菜单真相源继续稳固

目标：

- 数据库菜单即正式值
- 前端只保留正式映射
- 避免后续继续叠加历史兼容

### 4. 标准页继续复用化

目标：

- 后续新增业务页直接套现有公共层
- 减少重复写筛选区、按钮权限、详情抽屉、弹窗布局

### 5. 为财务预算模块准备平台接入条件

首个真实业务模块接入前，应优先补齐：

- 模块注册 / 模块入口配置
- 审批流 MVP
- 我的待办 / 我发起的
- 附件中心
- 统一业务操作日志

并控制在第一期最小闭环内，不扩展到流程设计器、消息中心、数据权限等二期能力。

## 当前不建议优先做的

- 数据权限二期
- 表单构建原生重写
- 代码生成危险写入动作开放
- 大范围视觉重构
- 在审批流和待办能力缺失的情况下直接上复杂财务业务

## 验收基线

当前收尾验收基线是：

- `go test ./...`
- `pnpm check:type`
- `pnpm build:local`
- `pnpm test:e2e`
