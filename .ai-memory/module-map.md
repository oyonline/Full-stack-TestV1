## 系统设置

- 正式入口：`参数设置`
- 职责：
  - 系统名称
  - Logo
  - Logo 占位色
  - 登录页标题/说明
  - 全局界面设置

## 字典管理

- 正式主入口：`字典类型目录页`
- 正式维护入口：`字典类型详情页`
- 当前职责拆分为：
  - 目录页：
    - 查找类型
    - 浏览类型
    - 新增类型
    - 删除空类型
    - 进入详情
  - 详情页：
    - 展示当前类型摘要
    - 展示“当前类型下共 N 条字典数据”
    - 在当前类型上下文里新增 / 编辑 / 删除字典数据
- `全部字典数据` 独立页已废弃，不再作为模块级正式入口保留。

## 权限与用户

- 用户支持多角色
- 菜单按主角色生效
- 按钮权限按并集生效
- 接口权限当前按主角色 key 走 Casbin 校验
- 数据权限本期关闭

## 后台标准页

公共层已具备：

- 列表页底座
- 树形页底座
- 详情抽屉
- 权限按钮
- 轻量弹窗 schema

## 头像与个人入口

- 右上角头像下拉当前已收口为真实用户入口：
  - 个人中心
  - 问题&帮助（占位）
  - 锁屏
  - 退出登录
- 当前 working tree 中存在头像相关本地方案，但正式 migration 仍未进入已提交链。
- 因此头像这条线目前只能按“本地待收口方案”理解，不能视为仓库主线完成模块。
- 个人中心“基本设置”当前拆成两块：
  - 只读资料块：姓名、登录账号、主角色、全部角色
  - 自维护资料块：个人简介
- 个人中心当前不承担角色、部门、岗位、权限编辑能力
- 个人中心当前已落地的真实自维护能力为：
  - 修改密码
  - 修改个人简介

- 当前后端业务实验层已基本收口，clone 基线本身已包含同事融合进来的业务代码。
- 当前更适合继续推进的方向是：
  - 前端原型补全
  - 页面承接与交互完善
  - 样式与体验收口
  - 脚手架增强
  - 文档同步
- 当前不建议继续把 finance、feishu、kingdee、biz_action_log、`sys_user` 扩展类本地后端实验重新带回主线。
