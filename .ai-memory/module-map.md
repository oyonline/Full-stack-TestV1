# 模块地图

## 分层视角

当前项目建议按两层看：

- 平台底座层：
  - 登录认证
  - 用户 / 角色 / 部门 / 岗位
  - 权限体系
  - 菜单与导航
  - 系统参数
  - 字典
  - 日志审计
  - 上传下载
  - 前后端统一协议
- 平台能力层：
  - 代码生成
  - 表单构建
  - 审批流管理（Step 1 后端已落地）
  - 待办中心（Step 2 最小验收层已落地）
  - 通知中心（当前仅保留右上角入口占位，未接真实数据）
  - 附件中心（Step 3 最小版已落地）
  - 后续通用单据状态流转

详细判断见：[platform-layer-audit.md](/Users/linshen/Desktop/Full-stack-TestV1/.ai-memory/platform-layer-audit.md)

## 系统设置

- 正式入口：`参数设置`
- 职责：

  - Logo
  - Logo 占位色
  - 登录页标题/说明
  - 全局界面设置

## 权限与用户

- 用户支持多角色
- 菜单按主角色生效
- 按钮和接口权限按并集生效
- 数据权限本期关闭

## 后台标准页

公共层已具备：

- 列表页底座
- 树形页底座
- 详情抽屉
- 权限按钮
- 轻量弹窗 schema

## 头像与个人入口

- 右上角头像下拉当前已收口为真实用户入口：
  - 个人中心
  - 问题&帮助（占位）
  - 锁屏
  - 退出登录
- 默认头像采用：
  - 自定义头像优先
  - 否则使用“用户名首字母大写 + 账号级稳定纯色背景”
- 个人中心后续承接“自定义头像 / 默认背景色”设置能力
- 个人中心“基本设置”当前拆成两块：
  - 只读资料块：姓名、用户名、主角色、全部角色
  - 自维护资料块：个人简介
- 个人中心当前不承担角色、部门、岗位、权限编辑能力
- 个人中心当前已落地的真实自维护能力为：
  - 修改密码
  - 修改个人简介
