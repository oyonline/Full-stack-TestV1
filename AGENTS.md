# AGENTS

## 1. 项目定位与仓库结构
- Full-stack-TestV1 是前后端同仓的全栈脚手架工作区，不是单体服务仓库。
- `go-admin` 是 Go + Gin 后端，承载 API、权限、迁移、种子数据和平台能力。
- `vue-vben-admin` 是 Vue 3 + Vben 前端 monorepo，主应用是 `apps/web-antd`。
- 前端当前采用 `backend` access mode；业务菜单不是前端硬编码，来自后端菜单树。
- 后端主业务层按 `app/admin`、`app/platform`、`app/jobs` 等模块组织。

## 2. 固定工作方式
- 每次开始执行任务前，先用中文输出：
  1. 本次改动的大目标
  2. 当前情况和任务进度
  3. 后续任务概览
- 一次只给一步操作建议，避免并行抛出多个下一步。
- 先分析，再改动；先确认真实约束和落点，再进入实现。
- 默认优先最小改动，不做无关重构，不顺手扩散治理。
- 需要新增依赖时，先确认，再执行。
- 如果 memory、文档与当前代码冲突，以当前代码为准，并明确指出冲突点。
- 默认目标是基于现有后端能力完善真实页面与真实链路，不做 mock-based page。
- 涉及本地联调、依赖、检查方式时，参考 `docs/local-dev-handoff.md`。
- 会长期影响协作的结论，必须同步回写 docs 或 `.ai-memory/*`，不能只留在对话里。

## 3. 真相源与冲突裁决顺序
- 当前任务的验收标准仅约束本次任务执行；若结论会长期生效，必须同步修改对应真相源代码或文档，不能只停留在本次说明中。
- 代码真实行为与正式迁移链，优先于说明性文档。
- 长期规则与协议，优先读取 `.ai-memory/*` 中的正式条目。
- 本地联调事实与当前运行口径，以 `docs/local-dev-handoff.md` 为准。
- 其余 README、历史 SQL、手工修库脚本只作补充线索，不单独作为最终裁决依据。

项目内固定真相源：
- 菜单与业务路由真相源是数据库 `sys_menu`；前端路由映射层只负责映射和少量兼容。
- 运行时系统设置真相源是后端配置与相关接口；前端本地状态不是正式配置源。
- 长期保留的 schema / 数据修复真相源是 `go-admin/cmd/migrate/migration/version/*`；`config/db.sql` 是初始化基线，`docs/sql/*` 和 `menu-batch*.sql` 只是参考或人工修复入口。
- 当前多角色真实行为以代码为准：菜单按主角色，按钮权限按并集，后端 Casbin 接口校验按主角色 key。

## 5. 高风险约束
- 不要把菜单真相源从数据库搬回前端硬编码。
- 不要在没有 migration 或配套 SQL 的前提下批量改 `sys_menu.path`、`component`、`permission`。
- 不要只改多角色链路中的一层；菜单、`getinfo`、Casbin 校验必须联动审视。
- 不要把历史兼容逻辑散落到业务页面；兼容层应收口在路由映射或显式隐藏路由。
- 不要批量重命名历史权限码；现有权限码已经进入菜单、按钮和 Casbin 链路。
- 不要把需要长期生效的 schema / 数据修复只留在 `docs/sql/*` 或 `menu-batch*.sql`。
- 不要为普通新接口绕过统一 API 层，直接在页面里拼请求。
- 不要默认引入新依赖来解决本可在现有栈内完成的问题。
- 不要为了赶页面而脱离现有后端能力做纯 mock 页面闭环。

## 6. 任务最小信息包
处理任何任务时，至少先明确下面 6 件事：
- 变更范围：前端、后端、数据库、迁移、文档，分别是否涉及。
- 目标对象：页面、菜单、接口、数据表、权限点、迁移版本，具体是哪一个。
- 真相源：这次改动最终应落在哪个真相源上，不能只改兼容层或展示层。
- 风险链路：是否影响登录、菜单、权限、配置、代码生成或本地初始化。
- 验收方式：本次最小可验证闭环是什么，是否需要联调或服务支持。
- 文档回写：这次结论是否需要同步到 `docs/local-dev-handoff.md` 或 `.ai-memory/*`。

## 7. Done 定义
- 相关代码、迁移、配置和文档已经在同一口径下闭合，没有把长期规则留在临时说明里。
- 变更已经落到正确真相源，而不是只改表象层或兼容层。
- 对高风险链路的影响已被验证，或已明确记录未验证原因。
- 需要长期保留的结论，已经同步到 `docs/local-dev-handoff.md` 或 `.ai-memory/*`。
- 仓库没有因为本次任务新增新的硬编码菜单真相、临时权限补丁或只对当前任务有效的一次性规则。
- 页面、接口和数据流已基于现有真实后端能力打通；若未打通，原因已明确记录。
