# 决策记录:SKU 数据权限策略(选项 1)

> 状态:已决策 | 日期:2026-05-09 | 决策人:产品(EPO-69) | 落地承接:架构师(EPO-83)
> 关联:EPO-69(决策)、EPO-73(后端落地清单)

## 1. 决定的选项

**选项 1 — SKU 主管理页只读 + 写动作走 SPU 编辑页;SKU 列表 LEFT JOIN spu 继承 SPU 的 dataScope。类目 / 品牌不接 dataScope。**

具体形态:

- **SKU 主管理页**:仅作只读浏览/查询入口。新增 / 编辑 / 删除入口移除,改由 SPU 编辑页内的 SKU 子表完成。
- **SKU 列表查询**:`sku LEFT JOIN spu ON sku.spu_id = spu.id`,dataScope 过滤条件作用在 `spu.creator_id` / `spu.dept_id` 上,SKU 表本身**不**新增 `creator_id` / `dept_id` 字段。
- **类目(category)/ 品牌(brand)**:维持公司级共享词典,**不接** dataScope,任何角色按角色权限可见/可写,不按部门隔离。

## 2. 理由

1. **业务侧无 SKU 主页独立写动作的真实诉求。** 当前没有 ERP 对接、秒杀、SKU 维度批量操作等场景;SKU 是 SPU 的子单元,孤儿 SKU(无所属 SPU)在业务上无意义。把"写"统一收敛到 SPU 编辑页,概念更清晰。
2. **品牌 / 类目是公司级共享词典。** 按部门隔离会造成"A 部门的 Apple"与"B 部门的 Apple"分裂、库内同义词膨胀,数据治理崩塌。这两类资源的控制点应是**角色权限**(谁能编辑词典),不是**数据权限**(谁能看到哪些行)。
3. **改动成本显著低于选项 2。**
   - 选项 1:不动 schema、不需要回填、不引入业务模型扭曲(SKU 仍是 SPU 子单元),后端只做 LEFT JOIN 改写 + 前端入口收敛。
   - 选项 2:需要 `sku` / `category` / `brand` 三表 ALTER 增加 `creator_id` / `dept_id` 字段,并对存量数据回填(已删用户场景需 fallback 策略),迁移风险与回滚成本都显著更高。

## 3. 影响范围

- **后端**:SKU 列表查询改 LEFT JOIN spu;dataScope 过滤条件迁到 spu 字段;移除 / 屏蔽 SKU 主页的写接口。
- **前端**:SKU 主管理页移除"新增 / 编辑 / 删除"入口,文案与跳转引导改为"请到 SPU 编辑页操作"。
- **数据**:无 schema 变更、无数据回填。
- **测试**:补 dataScope 5 路口径(1 全部 / 2 自定义 / 3 本部门 / 4 本部门及以下 / 5 仅本人)在 SKU 列表上的回归用例;补"SKU 主页写入口已下线"的负向用例。
- **文档**:更新 `docs/sku-module-guide.md` 关于写入口的章节;在 `PROJECT_CONVENTIONS.md` 数据权限章节登记"SKU 通过 SPU 继承 dataScope,不持有自身字段"的约束。
- **用户手册**:运营操作路径变更,SKU 写动作引导到 SPU 编辑页。

详细落地清单见 [EPO-73](mention://issue/808a619c-7d7d-49a2-a90a-ee16885552bf)。

## 4. 回滚条件

任一触发即重新走 RFC 启动选项 2 迁移:

1. 运营 1 个月内 ≥3 次反馈"必须 SKU 维度独立运营"高频诉求(例:批量改价、批量上下架按 SKU 而非 SPU 粒度)。
2. 合规升级要求 SKU 强隔离(例:GDPR / 行业监管要求 SKU 级别的部门数据可见性边界)。
3. 品牌 / 类目按部门隔离的真实业务场景出现(例:多事业部并购后短期内品牌词典需要分裂)。

## 5. 回滚成本预估

| 角色 | 预估工作量 | 主要内容 |
| --- | --- | --- |
| 后端 | 3d | sku / category / brand 三表 ALTER + creator_id/dept_id 回填 + dataScope 过滤切换 + 已删用户 fallback |
| 前端 | 1.5d | SKU 主页恢复写入口、品牌 / 类目页接 dataScope 显示逻辑 |
| 测试 | 1d | 三表 dataScope 5 路口径回归 + 迁移验证 |

合计约 5.5d 工程量,另需:数据迁移演练时间、灰度发布窗口、用户手册回滚版本。
